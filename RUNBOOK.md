# Runbook (эксплуатация)

Операционные процедуры и плейбуки на инциденты (NFR-7).

## Доступ к панели

```bash
ssh -L 8000:127.0.0.1:8000 vpnadmin@<vps-ip>   # держать открытым в терминале
# браузер -> http://127.0.0.1:8000 ; либо vpnctl общается с ней напрямую
```
Панель слушает localhost на сервере и **не** доступна из интернета по замыслу
(О-3). Если `curl http://<vps-ip>:8000` соединяется — это ошибка конфигурации,
остановись и разберись.

## Добавить / управлять пользователями

```bash
vpnctl users add bob --traffic 100 --expire 2026-12-31   # ссылка + QR
vpnctl users list
vpnctl users disable bob        # enable / delete / reset-traffic аналогично
```

## Инцидент: IP недоступен из России

**Симптом:** `vpnctl monitor ru` показывает 0/N доступных RU-нод, или клиенты в РФ
перестают подключаться, хотя сервер в остальном здоров.

1. Убедись, что сам сервер жив: `vpnctl status`, `vpnctl monitor health`.
2. Убедись, что это блок со стороны РФ, а не простой: достучись до `443` с не-РФ хоста.
3. Если блок из РФ → **ротация IP:**
   ```bash
   vpnctl users export .build/users-export.json   # через текущий туннель
   vpnctl rotate-ip
   ```
   Открой туннель к новой панели, когда попросят, чтобы пользователи
   импортировались. Если `MARZBAN_PANEL_DOMAIN` задан и его DNS перенаправлен на
   новый IP — клиенты подтянут новый сервер сами; иначе разошли subscription-ссылки заново.
4. Если ротация у того же провайдера продолжает попадать в бан — смени
   `VPS_PROVIDER` (UpCloud ⇄ Scaleway) в `.env` и ротируй снова.

## Инцидент: «заморозка» TLS (коннект есть, данные встают на ~15–20 КБ)

**Симптом:** `vpnctl monitor freeze` показывает `connected=True frozen=True`;
клиенты делают хендшейк, но страницы висят.

1. Это симптом шейпинга ТСПУ — IP/маршрут под подозрением. Реагируй как на бан IP:
   ротируй IP (см. выше).
2. Если повторяется быстро на свежих IP — включи режим маскировки WS+CDN (COULD,
   см. ARCHITECTURE) или перенеси основной канал на `separate-host` AWG на время разбора.

## Инцидент: сервис упал

**Симптом:** `vpnctl status` → «Marzban unreachable» (при открытом туннеле).

1. На сервере: `cd /opt/marzban && docker compose ps` и `... logs --tail=100`.
2. `docker compose up -d`, чтобы поднять обратно; у контейнера `restart: always`,
   так что один только ребут не должен его ронять — проверь диск (`vpnctl monitor health`).
3. Конфиг Xray сломан после правки? Проверь до рестарта:
   `docker compose exec marzban xray -test -c /var/lib/marzban/xray_config.json`.

## Инцидент: Reality спалили в регионе

**Симптом:** Reality-соединения не идут по всему региону, хотя IP доступен и сервис
работает.

1. Ротируй ключи Reality и SNI без пересоздания сервера:
   ```bash
   vpnctl sni check                     # выбрать свежий маскировочный домен
   $EDITOR .env                         # обновить REALITY_SNI; очистить REALITY_* ключи для регенерации
   vpnctl configure && vpnctl deploy    # выкатывает новый xray_config.json
   ```
2. Тем временем выдай затронутым пользователям резервный конфиг AmneziaWG.

## Клиент не подключается

1. Актуальна ли subscription-ссылка? Перевыпусти: `vpnctl users list`, затем перешли.
2. Пользователь превысил лимит / истёк срок? `vpnctl users reset-traffic <name>` или
   пересоздай с новым `--expire`.
3. Сбитое время на клиенте ломает Reality — проверь часы устройства.
4. Попробуй резервный профиль AWG, чтобы изолировать проблемы, специфичные для Reality.

## Бэкапы и восстановление

```bash
vpnctl backup                          # зашифрованный архив в BACKUP_DIR
vpnctl restore backups/vpn-backup-<ts>.tar.gz
```
Периодически проверяй восстановление (NFR-3): восстанови во временную папку и сверь
полученные `xray_config.json` / экспорт пользователей с продакшеном.

## Рутина

- Таймеры (см. `monitoring/`) гоняют healthcheck (5 мин), RU-check (10 мин), бэкап
  (ежедневно). Алерты уходят в Telegram, если настроен.
- Держи версии Xray/Marzban свежими, но запиненными — подними `XRAY_VERSION` /
  `MARZBAN_VERSION` в `.env`, затем `vpnctl deploy`. **Не** включай PQ-TLS / новые
  Vision-seed дефолты (О-4).
